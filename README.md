# Система управления лифтами в многоквартирном доме

Многопоточная система симуляции работы нескольких лифтов с оптимальным распределением запросов между ними.

## Описание проекта

Система имитирует работу M лифтов в доме с N этажами. Каждый лифт работает в отдельном потоке, диспетчер распределяет запросы пассажиров между лифтами по выбранной стратегии. Система обеспечивает потокобезопасность, логирование всех событий и сбор статистики.

## Классы:
1. **Direction** - перечисление направлений движения (UP, DOWN, NONE)
2. **ElevatorStatus** - статусы лифта (MOVING, STOPPED, DOORS_OPEN, EMERGENCY, MAINTENANCE)
3. **RequestType** - типы запросов (CALL, TARGET, PRIORITY, EMERGENCY)
4. **PassengerRequest** - запрос пассажира (этаж вызова, направление, целевой этаж)
5. **Elevator** - класс лифта (работает в отдельном потоке)
6. **ElevatorController** - диспетчер (работает в отдельном потоке)
7. **Main** - точка входа, запуск симуляции

## Дополнительные функции:

- Collective Control - оптимизация маршрутов с учетом времени ожидания, энергоэффективности и загруженности
- Ограничение вместимости - каждый лифт имеет максимальную вместимость
- Приоритетные этажи - дополнительное время ожидания на важных этажах
- Аварийный режим - немедленная эвакуация к ближайшему этажу
- Режим техобслуживания - временное отключение лифтов


# Статистика

По завершении симуляции выводится детальная статистика:

- Общее количество запросов и успешно обработанных
- Среднее время обработки запроса
- По каждому лифту:
    - Текущий этаж и статус
    - Количество перевезенных пассажиров
    - Проеханные этажи
    - Время простоя
    - Эффективность (этажи/пассажир)
